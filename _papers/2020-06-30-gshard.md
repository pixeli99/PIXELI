## 一句话

把 sparse MoE 接进 Transformer encoder，配合一套基于 XLA 的 `replicate / split` 标注 API，在 2048 块 TPU v3 上训出 **600B 参数**的多语言翻译模型。

## 方法

- 在 Transformer encoder 的偶数层把 FFN 替换为 MoE 层，**top-2 gating**，2048 个 expert 切到 2048 个 device 上，每个 expert 一份
- 引入 `expert capacity = (tokens_per_batch / num_experts) * capacity_factor`（典型 1.0–2.0），超出部分 token 直接 drop（不经 expert，只走 residual）
- 加 `Random routing` 二选一：第二选以概率 $p \propto g_2$ 取，否则视为 top-1，省 all-to-all
- aux loss 用 $\ell_{\mathrm{aux}} = N \sum_i f_i \cdot P_i$，$f_i$ 为分到 expert $i$ 的 token 比例，$P_i$ 为 gate 概率均值，这条公式一直被沿用
- 分片用 `mesh_split(tensor, partition_spec)` 注解，compiler 自动插 all-to-all / all-reduce；600B 模型 4 天训完 1T tokens，100 个语言到英文 BLEU 全面提升

## 想法

- 真正可复现的 sparse MoE 模板就是从这里开始：top-2 + capacity factor + token dropping + load-balancing aux-loss
- 标注式分片是后来 GSPMD / PaLM 的雏形，比手写 pipeline parallel 干净太多。不过 capacity factor 是个糟糕的超参，太小掉点太大浪费显存，后来 Expert Choice 就在这点上反攻
- 600B 这个数字在 2020 年看是炸的，但其实 99% 参数在 expert 里，单 token FLOPs 远低于同参数稠密模型

## 一句话

一套固定超参 + 一套架构在 150+ 任务（DMC、Atari、ProcGen、Crafter、Minecraft、DM Lab）上稳定 work，最大亮点是 Minecraft 中从零学会挖钻石（无 demo、无 curriculum），并展示 world model 参数 8M→200M 时性能单调上升。

## 方法

- 沿用 RSSM + discrete latent（V2）+ actor-critic in imagination
- 关键稳定性 trick：
  - **symlog 预测**：$\mathrm{symlog}(x) = \mathrm{sign}(x)\log(|x|+1)$ 把 reward/value 压到可控范围，去掉 reward clipping
  - **two-hot encoded** discrete regression head 取代 MSE
  - **percentile return normalization**（5–95 分位）让 actor entropy bonus 跨任务可比
  - KL balancing 沿用 V2，自由比特 $\geq 1$ nat 防 posterior collapse
- Minecraft diamond：单一脚本、200M env steps，约 17 天单 GPU，成功率 $\approx 2\%$（首个 from-scratch 不靠 demo 的结果）
- Crafter score 14.77，远超之前 SOTA；Atari 100k mean 1.12

## 想法

- "一套超参跨 150 任务"是个真正有分量的承诺，做过 RL 调参的人都知道这个数字背后意味着多少 sanity check
- symlog + two-hot 看似工程小修小补，但解决的是 reward magnitude 跨域不可比这个根本问题；这套组合后来直接被 TD-MPC2 / SimBa 抄走
- diamond 的 2% 成功率不高，但意义在于它把 model-based 推到 partially observable + sparse reward + long horizon 的硬场景；Voyager 这种 LLM agent 派看不上 RL 派，这篇是 RL 派的反击
- scaling 曲线只到 200M，比起 LLM 远远不够；真正的 world model scaling law 还得等更后面

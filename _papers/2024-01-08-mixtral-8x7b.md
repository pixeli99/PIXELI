## 一句话

8 个 expert / top-2 / 无 shared expert 的 decoder-only sparse LM，**46.7B 总参 / 每 token 激活 12.9B**，在多数 benchmark 上压 Llama-2-70B 和 GPT-3.5，权重 Apache-2.0 开源，开启了"小 expert 数高激活比"的 MoE 路线。

## 方法

- 架构：32 层 Transformer，每层 FFN 换成 8 expert 的 MoE，gate 用 top-2，无 shared expert
- 路由公式 $y = \sum_{i \in \mathrm{TopK}(g(x), 2)} \mathrm{Softmax}(g(x))_i \cdot E_i(x)$，gate 是一个普通 linear
- 上下文 32k，SwiGLU，RoPE，GQA（8 KV head）；总参 46.7B（不是 56B，因为 attention / embed 共享），active 12.9B
- 数据多语言，预训练 token 数未披露
- benchmark：MMLU 70.6 / GSM8K 58.4 / HumanEval 40.2，分别超 Llama-2-70B 的 69.8 / 56.8 / 29.3，推理时只跑 13B 参数

## 想法

- "8 expert top-2"是 Mixtral 立下的范式，与 Switch / GLaM 的 64+ expert 路线截然不同。expert 数少 → 每个 expert 学得充分 → fine-tune 稳，部署也容易（8 个 expert 可以塞进单机 8 GPU 一卡一 expert）
- 无 shared expert 这点后来被 DeepSeek-MoE 反对：DS 论证 shared expert 能吸走 common knowledge 让 routed expert 更 specialize。Mixtral 走的是另一极
- 工程影响远大于算法影响：vLLM、TGI、llama.cpp 都立刻支持，社区第一次有可用的开源 MoE，megablocks / HF transformers 集成是从这篇之后才真正爆发
- 论文写得极简，几乎不谈训练细节（aux-loss、z-loss、capacity factor 都没说），跟同期 OLMoE 的全开源形成对比

## 一句话

MLA 把 KV cache 砍到原来的 6.7%，再叠 DeepSeekMoE，造一个 236B 总参 / 21B 激活的便宜大模型。

## 方法

两个核心组件：

- **Multi-head Latent Attention (MLA)**：把 KV 投到一个低秩 latent $c^{KV}_t \in \mathbb{R}^{d_c}$，推理时只缓存 $c^{KV}_t$（外加一小段 decoupled RoPE 用的 $k^R_t$），上投到每头 $K, V$ 的矩阵可以吸收进 $W^Q$ 和输出矩阵里，所以推理实际不"上投"。相比 MHA，KV cache 体积下降 93.3%。
- **DeepSeekMoE**：沿用 2401.06066 的细粒度 + shared expert，160 个 routed expert + 2 个 shared，top-6 routed。Device-limited routing 限制单 token 跨机数。

8.1T tokens 预训练，context 128K（YaRN 扩展）。生成吞吐相对 DeepSeek 67B 提升约 5.76 倍。Aux loss 上加了一个 device-level balance 和 communication balance。

## 想法

MLA 是这篇真正的发明，等价于把 KV 从"每头一份"折成一个共享 latent，再让 head 各自 lift。它聪明的地方是把 RoPE 拆出来 decouple，不然 RoPE 的相位旋转和 latent 上投不兼容——这一步常被人忽略。

缺点：MLA 的实现细节（怎么吸收 $W^{UK}$、怎么和 FlashAttention 配合）论文写得偏粗，复现的人吃了不少苦头。Aux loss 一堆，调起来烦——这条路在 2408 的 Loss-Free Balancing 里被自己推翻。下一步该看：MLA 在长上下文外推时质量曲线是否真的优于 GQA。

## 一句话

完全开源（权重 + 数据 + 训练代码 + checkpoint + log）的 sparse LM，**6.9B 总参 / 每 token 激活 1.3B**，5T tokens 预训练后在同 active 参数级别打过 Llama-2-13B / DeepSeek-MoE-16B / Qwen-1.5-MoE-A2.7B。

## 方法

- 64 expert / top-8 / 无 shared expert（与 Mixtral 反向：更多 expert，更高 top-K，但激活比更小）
- dropless 训练（基于 megablocks），load-balancing loss 权重 0.01，router z-loss 权重 0.001
- 5T tokens（DCLM + Dolma 1.7 子集），AdamW，bs 4M tokens，cosine LR
- benchmark：MMLU 54.1，GSM8K 40.9，HumanEval 36.0，参数效率显著高于稠密 1.3B
- 全套 ablation 公开：shared vs no-shared expert（无 shared 好 ~0.5 pp）、top-K 扫描（8 是甜区）、expert 数 8 vs 64（64 更好但 diminishing）、dropless vs token-drop（dropless +0.3 pp）

## 想法

- 当时（2024 H2）唯一一个"全开源训练栈"的 MoE，Mixtral 只开权重、DeepSeek 开权重 + 部分细节，OLMoE 把训练数据、optimizer state、中间 checkpoint 一锅端。学术价值远高于 benchmark 数字
- 64 expert top-8 这个配置其实是回到 Switch / ST-MoE 时代的甜区，证明 Mixtral 的 8-expert 不是必然，可能只是 Mistral 出于部署考虑的选择
- ablation 那张表是这两年最有用的 MoE 实验文档之一：shared expert 这件事它的结论与 DeepSeek-MoE 相反，值得对照看（差异可能在 expert 数与 top-K 的耦合上）
- 训练 log 公开后社区可以直接复现 spike 出现位置和恢复方式，这种透明度才是 open source 该有的样子

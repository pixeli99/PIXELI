## 一句话

在 V3-Base 上直接大规模 RL，不靠 SFT 也能逼出长 CoT 推理（R1-Zero）；再加一层 cold-start SFT + 多阶段 RL 解决可读性，得到 R1。

## 方法

架构完全沿用 DeepSeek-V3（671B/37B MoE），不动。变量只在 post-training。

- **R1-Zero**：V3-Base → 直接 RL，奖励只看两件事：答案正确性（规则匹配，数学/代码 verifier）+ 输出格式（`<think>...</think><answer>...</answer>`）。RL 算法用 **GRPO**——去掉 critic，用同 prompt 一组 sample 的奖励均值/方差当 baseline。训练中模型自发产生"反思"、"重新评估"、aha moment（论文专门有图）。AIME 2024 pass@1 从 15.6% 提到 71.0%，majority voting 86.7%。
- **R1**：解决 Zero 的 mixing 语言、可读性差。流水线：
  1. 小批 cold-start CoT 数据 SFT
  2. 推理任务 RL（同 Zero 的奖励 + 语言一致性奖励）
  3. 拿 RL checkpoint 做 rejection sampling 生成新 SFT 数据，混入通用任务，再 SFT
  4. 全场景 RL（推理 + helpfulness + harmlessness）
- **Distillation**：把 R1 生成的数据 SFT 到 Qwen、Llama 系列小模型，1.5B–70B 全开。

## 想法

R1-Zero 是这篇最有冲击力的部分——它给出了 verifier-only RL 的下限：只要 reward 信号足够干净，base model 足够强，CoT 长度会自己涨，正确率随之涨。这件事之前都假设需要 process reward 或 SFT bootstrap，R1-Zero 证明这两个都不是必需。

可质疑：reward hacking 的痕迹其实是有的（输出风格收敛、某些题型靠模板），论文用了语言一致性奖励来 patch，相当于承认纯 outcome reward 在生产环境不够。另外 GRPO 相对 PPO 的优势在大模型上有多稳，paper 没仔细 ablation。

下一步该看：R1 风格的训练能不能往非可验证任务（写作、agent 规划）外推，以及 distillation 之后小模型的"反思行为"是真泛化还是模板模仿。

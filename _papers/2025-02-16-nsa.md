## 一句话

把 sparse attention 拆成 compression / selection / sliding 三条可训练分支并联，端到端训练，长上下文 prefill 和 decode 都加速。

## 方法

对每个 query $q_t$，三路 KV 同时算，再用一个 learned gate 加权融合：

- **Compressed branch**：把 history KV 按 block size 32（stride 16）压缩成 block-level 表示（带可学习的位置编码 MLP），$q_t$ 对压缩后的稀疏序列做 attention——拿"粗粒度全局"。
- **Selected branch**：用 compressed branch 的注意力分数当 importance score，选 top-$n$ 个 block（block size 64，count 16），在原 token 粒度上做 attention——拿"细粒度局部精度"。
- **Sliding branch**：固定窗口的近邻 attention——保住 recency。

三路输出 $o^c, o^s, o^w$ 用 sigmoid 门控 $g^c, g^s, g^w$ 线性混合。整套都是可微的，从 scratch 一起训。

工程上：

- 用 GQA / MQA 共享 KV，把同一 KV-group 内的 query 当成一个 block 处理，避免 sparse attention 在 decode 时"每个 head 算自己的稀疏 pattern"带来的访存放大。
- 三路都写了 Triton kernel，64K 长度下 forward / backward / decode 全段加速，且性能在 general / long-context / reasoning 多组 benchmark 上不输 full attention，部分还反超。

## 想法

亮点是把 sparse attention 从"推理期的近似"提到"训练期的一等公民"。以前 sparse attention（Longformer、BigBird 那一脉）的痛点是和 dense 训出来的模型不兼容，强行 swap 会掉点；NSA 直接 train-from-scratch 解决这件事。

selection branch 复用 compression branch 的 score 也很经济——省掉一个独立的 importance estimator，但代价是 selection 的精度上限被 compression 的容量绑死，这一点 paper 没正面讨论。

GQA-aware 的设计是 sparse attention 落地的关键工程细节，不然 decode 时 KV 读取放大会把稀疏带来的好处全吃掉。下一步该看：NSA 和 MLA 怎么并存（V3.2 给出了部分答案：直接走 indexer + selection 路线）。

## 一句话

1T 总参 / 32B 激活的 MoE，384 expert top-8（sparsity ≈ 48，比 DeepSeek V3 的 256/8=32 更激进），用自研 MuonClip 优化器在 15.5T token 上无 loss spike 完成预训练，瞄准 agentic 长程任务。

## 方法

- 架构：61 层，hidden 7168，每个 expert FFN 中间维 2048，1 个 shared expert + 384 routed expert top-8；attention 用 MLA（沿 DeepSeek V2 的 latent KV 压缩），64 head，vocab 160K，原生 128K context
- 优化器 **MuonClip** = Muon + QK-clip：Muon 用矩阵-orthogonalization 替 AdamW 的 second moment，token efficiency 高但在大 attention logits 下会炸，QK-clip 把 $W_q, W_k$ 的输出按 $\tau = 100$ 软截断，max attention logit 自然回到健康区间
- pre-train：4096 上下文窗口，WSD 学习率（10T const @ 2e-4，再 5.5T cosine 到 2e-5），weight decay 0.1，**global batch 67M token**；整个 15.5T 没有 observable loss spike
- post-train：大规模 agentic 数据合成 + joint RL，在真实和合成 tool-use 环境里训，强调 long-horizon coding / browsing
- 公开权重 + Apache-2.0 是这次的最大杀招

## 想法

- 真正值得抄的不是架构（架构基本是 DeepSeek V3 + 更激进 sparsity），是 **MuonClip**：Muon 在大模型上能用是这一年训练侧最大的开放问题，K2 给了第一个 1T 量级的实证。QK-clip 这种"在优化器层面治 attention 病"的做法很可能成新范式
- sparsity 48 这个数字工程上很挑：all-to-all 通信、expert imbalance、负载倾斜都会被放大。他们能跑下来说明 routing + capacity 的工程库已经成熟到一定程度
- "Open Agentic Intelligence" 这个标题是产品定位——不再争 MMLU，争 SWE-bench / tau-bench 这类 agent benchmark，post-train 的 RL pipeline 才是真护城河

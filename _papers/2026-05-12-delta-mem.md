## 一句话

给 frozen transformer backbone 挂一个 8×8 的关联记忆矩阵，用 delta rule 在线更新、低秩修正 attention 输出，不动主干 weight 就能做长期 memory。

## 方法

memory state 是一个固定大小矩阵 $M_t \in \mathbb{R}^{d \times d}$（论文里 $d=8$，所以只有 64 个数）。来一个新的 (key, value)：

$$
M_{t+1} = M_t + \eta \,(v_t - M_t k_t)\, k_t^\top
$$

这就是经典的 delta rule——Hebb 学习的修正版，跟最近 Gated DeltaNet 同根。query 时用 $\hat v = M k_q$ 出 memory 读数，作为对原 attention 输出的 **低秩 correction**：

$$
o = o_\text{attn} + W_o \,\hat v
$$

backbone 完全冻结，只训练 $\eta$、$W_o$ 和读写投影。

results（相对 frozen backbone）：

- 综合 1.10×
- 相对最强非 δ-mem baseline 1.15×
- MemoryAgentBench 1.31×
- LoCoMo 1.20×

## 想法

8×8 这个尺寸让人怀疑——64 个标量到底装得下多少有效信息？我猜真正起作用的是 $W_o$ 那层 projection 和 backbone 自身的 in-context 能力，memory matrix 更像是个"激活引导器"，提供低维状态让 attention pick 起对应 chunk。

跟 RWKV、Mamba、Gated DeltaNet 这一支线性 RNN 的区别在于：δ-mem 不替换 transformer，是 plug-in。这种"冻结 LLM + 外挂状态"的形态对生产环境很友好——不用重训 base model 就能上长 memory，跟 LoRA 的 deployment story 一致。

但 1.10× / 1.15× 的提升放在 backbone 已经 100B 参数面前显得边际。希望看到 32K 以外的、像 NIAH-1M 这种极端长上下文 task。MemoryAgentBench 1.31× 是最有说服力的数——agent 跨 session 记事本类任务，正好是 backbone 外挂记忆该发光的地方。

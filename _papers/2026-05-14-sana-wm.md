## 一句话

2.6B 开源 world model，原生为"1 分钟 720p"生成训练，hybrid linear attention + 6-DoF dual-branch camera control，蒸馏后在单卡 RTX 5090 + NVFP4 下 34 秒出 60 秒视频。

## 方法

模型主体是 SANA 风格的 DiT，关键改动：

- **Hybrid linear attention**：frame-wise 用 Gated DeltaNet（线性复杂度），帧间用 softmax attention。1 分钟 = 上千帧的 token 序列在全 softmax 下根本跑不动，这套 hybrid 是分钟级 generation 的关键
- **Dual-branch camera control**：6-DoF 相机轨迹通过独立的 condition branch 注入，避免和 text prompt 在同一个 cross-attention 里抢容量
- **两阶段生成**：base model 出粗版，long-video refiner 再过一遍提质量、补一致性
- **数据**：213K 公开视频 + 自家 metric-scale 6-DoF pose 标注 pipeline

训练 15 天 / 64 H100。部署侧 NVFP4 量化 + step distillation，60s/720p 在 RTX 5090 上 34 秒——比之前开源 baseline 吞吐高 36×。

## 想法

相比 Genie 3 / Marble 那种"闭源 + 演示驱动"的路线，SANA-WM 走的是"小、开源、能在消费卡跑"的极端。这个定位其实更接近 Stable Diffusion 时刻——能否触发社区二创取决于模型能不能 fine-tune。

linear attention + softmax 的 hybrid 这两年试过很多次，分钟级视频是个比较硬的 stress test，跑通了说明这条路在长视频上有戏。但他们没和 NSA / GLA 这种 sparse attention 对比，hybrid 比 sparse 强在哪并不清楚。

NVFP4 + 34 秒这种数字是 paper 的真正卖点——交互式 world model 必须卡到 sub-real-time。如果开源真把 checkpoint + 量化代码都放出来，会立刻成 community baseline。
